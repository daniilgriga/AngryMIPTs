# Руководство по созданию уровней AngryMipts

Этот документ описывает текущий JSON-контракт игры: какие поля должен содержать уровень, как задаются блоки и цели, как считаются очки и звёзды, какие значения HP и score лучше использовать.

> [!important]
> Runtime-источник уровня для игры — JSON-файл в формате, который понимает `LevelLoader`. Уровень можно собрать любым инструментом, если итоговый JSON соблюдает контракт ниже.

## Минимальная структура уровня

```json
{
  "meta": {
    "id": 1,
    "name": "level_01",
    "totalShots": 3,
    "starThresholds": [2000, 2500, 2990]
  },
  "slingshot": {
    "position": [140, 600],
    "maxPull": 195
  },
  "projectiles": [
    { "type": "Standard" },
    { "type": "Splitter" },
    { "type": "Heavy" }
  ],
  "blocks": [],
  "targets": []
}
```

Обязательные верхнеуровневые поля:

| Поле | Тип | Назначение |
|---|---:|---|
| `meta` | object | ID, имя, количество выстрелов, пороги звёзд |
| `slingshot` | object | Положение и сила натяжения рогатки |
| `projectiles` | array | Очередь снарядов, строго по порядку |
| `blocks` | array | Разрушаемые или статичные блоки |
| `targets` | array | Цели, уничтожение которых нужно для победы |

## Метаданные

```json
"meta": {
  "id": 3,
  "name": "level_03",
  "totalShots": 5,
  "starThresholds": [3500, 4434, 5369]
}
```

Правила:

- `id` — положительное целое число.
- `name` — непустая строка. Для текущих уровней используется стиль `level_01`, `level_02`, ..., `level_010`
- `totalShots` должен точно совпадать с количеством объектов в `projectiles`.
- `starThresholds` — массив ровно из трёх строго возрастающих чисел.
- Звезда засчитывается, если итоговый счёт `>=` соответствующего порога.

> [!note]
> Победа принудительно даёт минимум 1 звезду, но пороги всё равно должны быть корректными. Не стоит рассчитывать на это как на способ балансировки.

## Координаты и размеры

Все координаты задаются в пикселях.

Игровое поле валидируется в пределах `1920 x 1080` для стартовых координат объектов:

- `position[0] > 0`
- `position[1] > 0`
- `position[0] <= 1920`
- `position[1] <= 1080`

Для блоков и целей `position` означает центр объекта. Для рогатки `position` означает базовую точку рогатки.

## Рогатка

```json
"slingshot": {
  "position": [140, 600],
  "maxPull": 195
}
```

Рекомендации:

| Поле | Рекомендуемое значение | Комментарий |
|---|---:|---|
| `position.x` | `120-420` | Чем левее рогатка, тем больше места для полёта |
| `position.y` | `600` | В текущих уровнях это высота земли установленная на уровне движка |
| `maxPull` | `120-200` | Больше значение даёт более сильный выстрел |

Не ставьте рогатку внутрь блоков или слишком близко к ним. Игрок должен видеть траекторию и иметь пространство для натяжения.

## Снаряды

```json
"projectiles": [
  { "type": "Standard" },
  { "type": "Dasher" },
  { "type": "Bomber" }
]
```

Доступные типы:

| Тип | Роль в уровне |
|---|---|
| `Standard` | Базовый снаряд для обычных уровней и обучения |
| `Heavy` | Сильнее пробивает конструкции |
| `Splitter` | Делится на дополнительные снаряды |
| `Dasher` | Ускоряется при активации |
| `Bomber` | Взрывное разрушение группы объектов |
| `Dropper` | Сбрасывает дополнительный снаряд вниз |
| `Boomerang` | Меняет движение после активации |
| `Bubbler` | Создаёт область воздействия вокруг себя |
| `Inflater` | Увеличивается и давит на окружение |

Договорённости по балансу:

- Простые уровни: `3` снаряда.
- Средние уровни: `4-5` снарядов.
- Большие уровни: `5-6` снарядов.
- Не выдавайте слишком много спецснарядов без причины: уровень становится шумным и хуже читается.
- Первый снаряд лучше делать понятным для основной задачи уровня: `Standard`, `Dasher`, `Bomber` или `Dropper`.
- Очередь снарядов должна быть частью решения уровня, а не случайным набором.

## Блоки

Каждый блок — это объект в массиве `blocks`. Набор обязательных полей зависит от формы блока.

Общие обязательные поля для любого блока:

| Поле | Тип | Для каких форм | Назначение |
|---|---:|---|---|
| `shape` | string | все | `rect`, `circle` или `triangle` |
| `material` | string | все | `Wood`, `Stone`, `Glass`, `Ice` |
| `position` | `[x, y]` | все | Центр блока |
| `angle` | number | все | Поворот в градусах |
| `hp` | number | все | Прочность объекта |
| `static` | bool | опционально | Неподвижный и неразрушаемый объект |
| `indestructible` | bool | опционально | Неразрушаемый объект |

Поля, зависящие от `shape`:

| `shape` | Обязательные геометрические поля | Ограничения |
|---|---|---|
| `rect` | `size: [w, h]` | `w > 0`, `h > 0` |
| `circle` | `radius` | `radius > 0`; `size` не нужен |
| `triangle` | `vertices` или `size` | Для новых уровней используйте `vertices`; если есть только `size`, загрузчик создаст legacy-шаблон треугольника |

Для `triangle` поле `size` можно не указывать, если есть корректный `vertices`: загрузчик сам вычислит bounding box по вершинам. Если указаны и `vertices`, и `size`, физика и рендер используют именно `vertices`, а `size` остаётся вспомогательным размером.

### Прямоугольный блок

```json
{
  "shape": "rect",
  "material": "Wood",
  "position": [950, 505],
  "size": [200, 20],
  "angle": 0,
  "hp": 50
}
```

### Круглый блок

```json
{
  "shape": "circle",
  "material": "Stone",
  "position": [950, 465],
  "radius": 28,
  "angle": 0,
  "hp": 110
}
```

### Треугольный блок

```json
{
  "shape": "triangle",
  "material": "Stone",
  "position": [984, 556],
  "angle": 0,
  "hp": 50,
  "size": [56, 56],
  "vertices": [
    [-28, 28],
    [28, 28],
    [0, -28]
  ]
}
```

Правила для `vertices`:

- ровно 3 точки;
- каждая точка — массив из двух чисел;
- точки задаются относительно центра блока, а не относительно мира;
- треугольник не должен быть вырожденным: точки не должны лежать на одной прямой;
- `angle` применяется уже после локальной геометрии.

> [!tip]
> Для новых уровней лучше всегда указывать `vertices` у треугольников. Legacy-вариант через один `size` всё ещё загружается, но даёт шаблонный равнобедренный треугольник.

## Материалы, HP и очки за разрушение

Материал и HP — разные вещи:

- `material` определяет визуал, бонусные очки за разрушение и часть физического поведения;
- `hp` определяет фактическую прочность конкретного объекта.

Очки за разрушенный блок:

| Материал | Очки за блок | Физический смысл |
|---|---:|---|
| `Glass` | `20` | Хрупкий материал |
| `Ice` | `30` | Хрупкий/скользкий материал |
| `Wood` | `50` | Базовый строительный материал |
| `Stone` | `100` | Тяжёлый и прочный материал |

Рекомендуемые HP:

| Материал | Лёгкий блок | Обычный блок | Усиленный блок | Комментарий |
|---|---:|---:|---:|---|
| `Glass` | `20-35` | `35-50` | `50-70` | Должен ломаться легче дерева |
| `Ice` | `20-30` | `30-45` | `45-60` | В текущих уровнях почти не используется, держать ближе к glass |
| `Wood` | `9-25` | `50` | `55-70` | `9` подходит для очень лёгких release-блоков |
| `Stone` | `15-50` | `50-90` | `90-110` | `15` подходит для лёгких каменных плит, `90+` для тяжёлых элементов |

> [!warning]
> Не делайте HP пропорционально очкам за материал. Например, stone даёт `100` очков, но если всем stone-блокам поставить слишком высокий HP, уровень станет медленным и плохо разрушаемым.

### Static и indestructible

```json
{
  "shape": "rect",
  "material": "Stone",
  "position": [960, 620],
  "size": [300, 20],
  "angle": 0,
  "hp": 50,
  "static": true
}
```

Правила:

- `static: true` делает блок неподвижным.
- В текущей физике `static` и `indestructible` фактически схлопываются: статичный блок становится неразрушаемым, а неразрушаемый — статичным.
- Статичные блоки не считаются в потенциальный бонус разрушения.
- Используйте static для ландшафта, платформ, стенок и ограничителей, но не для основных целей разрушения.

## Цели

```json
{
  "position": [905, 475],
  "radius": 16,
  "hp": 22,
  "score": 1400
}
```

Обязательные поля:

| Поле | Тип | Назначение |
|---|---:|---|
| `position` | `[x, y]` | Центр цели |
| `radius` | number | Радиус цели |
| `hp` | number | Прочность цели |
| `score` | int | Очки за уничтожение цели |

Рекомендуемые значения:

| Тип цели | HP | Score | Когда использовать |
|---|---:|---:|---|
| Малая цель | `12-20` | `500` | Дополнительная или легко доступная цель |
| Обычная цель | `15-22` | `1000` | Основной стандарт |
| Ценная цель | `22-50` | `1400-2000` | Сложно защищённая или ключевая цель |
| Босс/витрина | `75+` | `3000-4000` | Только для специальных уровней |

Уровень считается выигранным, когда все цели уничтожены.

## Как считать звёзды

Итоговый счёт складывается из:

1. очков за уничтоженные цели;
2. очков за разрушенные блоки.

Текущая договорённость для автопорогов:

```text
targetScore = сумма score всех targets
blockBonusMax = сумма бонусов всех разрушаемых blocks
maxScore = targetScore + blockBonusMax

star1 = targetScore
star2 = targetScore + round(0.35 * blockBonusMax)
star3 = targetScore + round(0.70 * blockBonusMax)
```

Где бонусы блоков:

```text
Glass = 20
Ice   = 30
Wood  = 50
Stone = 100
```

> [!important]
> `starThresholds` должны быть строго возрастающими: `star1 < star2 < star3`. Если в уровне слишком мало разрушаемых блоков, формула может не дать три разных порога.

### Пример расчёта

Допустим, в уровне:

- 3 цели по `1000` очков: `targetScore = 3000`;
- 10 wood-блоков: `10 * 50 = 500`;
- 5 stone-блоков: `5 * 100 = 500`;
- 4 glass-блока: `4 * 20 = 80`.

Тогда:

```text
blockBonusMax = 500 + 500 + 80 = 1080
maxScore = 4080

star1 = 3000
star2 = 3000 + round(0.35 * 1080) = 3378
star3 = 3000 + round(0.70 * 1080) = 3756
```

В JSON:

```json
"starThresholds": [3000, 3378, 3756]
```

### Когда менять пороги вручную

Можно отклоняться от формулы, если уровень специально задуман иначе:

- `star1` ниже `targetScore`, если уровень должен давать 1 звезду даже за частичное прохождение. Это меняет смысл звёзд и должно быть осознанным решением.
- `star3` выше `70% blockBonusMax`, если хотите требовать почти полную зачистку.
- `star2` можно сместить ближе к `star1`, если уровень сложный и хочется щадящего среднего результата.

Базовая рекомендация: если нет сильной причины, используйте формулу `100% целей / +35% блоков / +70% блоков`.

## Практические правила баланса

### Хороший уровень

Хороший уровень обычно имеет:

- понятную первую цель;
- 2-5 целей;
- 3-5 снарядов;
- хотя бы один очевидный слабый элемент конструкции;
- возможность получить 1 звезду за уничтожение целей;
- возможность получить 3 звезды за аккуратное разрушение значимой части конструкции.

### Чего избегать

- Слишком много stone-блоков с высоким HP: уровень становится вязким.
- Слишком много спецснарядов сразу: игроку сложно понять, какой навык проверяется.
- `totalShots`, не совпадающий с длиной `projectiles`: такой JSON не загрузится.
- `starThresholds`, где два числа равны или идут не по возрастанию.
- Треугольники без `vertices` в новых уровнях.

## Краткая памятка

```text
Размер поля: 1920 x 1080 px
Формы блоков: rect, circle, triangle
Материалы: Wood, Stone, Glass, Ice
Очки за блоки: Glass 20, Ice 30, Wood 50, Stone 100
Звёзды: targetScore / +35% blockBonus / +70% blockBonus
Победа: уничтожены все targets
totalShots: равно количеству projectiles
```
